Getting started

This is the demo site for the Boxmodel internal framework, aimed at making is easier and quicker to start new projects and provide a level of consistency between projects.

Getting started with the framework is easy, simply use this page as a template, remove the content and your good to go.

This page provides a simple description as to how each part is built up and the syntax required.

Make sure you have SASS installed on your machine as this is a SASS based framework, all styles have been split into seperate SCSS files and included within styles.scss. There is a seperate content.scss file for all your site specific styles.

Typography

The default body font size is 16px (this can be easily changed in global-variables.scss) and is set using em, so any font size decalred is relative to its parent. The framework did intially use REM and a fallback of EM but at the time of writing there is a bug in Chrome when using percentage fontsize on the HTML element and REM on the body. The line-height is calculated using Chris Pearson's golden ratio he describes here. The idea is that the line height should be equal to the font size * 'the golden ratio' (1.61803398875). This ratio value is stored as a variable in global-variables.scss. Some interesting information about The Golden Number can be found here.

Headings (h1, h5) by default start at 35px, there is a mixin in type.scss that calculates the sizes for all headers based on 10% of the previous header's size to keep them relative to each other. This can be easily changed to match a design.

Responsive images

To save page size and load time it is reccommended to serve suitable sized images depending on the resolutions, the resolution size typically relates to the device they are viewing the web page on.

The previous method was to use the same image at all resolutions, this either means you are serving a large image at small resolutions or a small image at large resolutions resulting in a loss of quality.

Serving different sized images

Using the below method the browser works out which image to serve, with a fallback to those that don't support srcset. The numbers to the right of each src represent the images natural width.

<img srcset="img-full-large.jpg 1920w, img-full-medium.jpg 1280w, img-full-small.jpg 768ww"
	src="img-full-medium.jpg"
	alt="Responsive images" />

The above methods leaves the selection of which image to serve down to the browser, you can have more control over which image the browser shows by setting some size rules. The attribute sizes refers to the layout width that will be containing the image.

<img srcset="img-full-large.jpg, img-full-medium.jpg, img-full-small.jpg"
	sizes="100vw"
	src="img-full-medium.jpg"
	alt="Responsive images" />

The example above states that the image to choose from will fill 100% of the viewport. You can use any form of css length here. The sizes attribute value doesn't actualy size the image to that width, it is purely to help the browser pick out the most suitable image, a good way to look at it is 'When in the page the image will be about 'this big''.

You could even use media queries.

<img srcset="img-full-large.jpg, img-full-medium.jpg, img-full-small.jpg"
	sizes="(min-width: 1280px) 960px, 100vw"
	src="img-full-medium.jpg"
	alt="Responsive images" />

The example above states that if the viewport is above 1920px then the image used will be displayed at around 960px, else it will be 100% of the viewport width.
